## 📖 О боте

**Market-stats** — это мощный Discord-бот для отслеживания статистики маркетплейса Majestic RP. Бот предоставляет удобный интерфейс для просмотра цен, объявлений и аналитики по различным категориям товаров.

### ✨ Возможности

- 📊 **Статистика маркетплейса** — детальная статистика по 7 категориям
- 📋 **Объявления** — просмотр актуальных объявлений с пагинацией
- 🔑 **API ключи** — безопасное хранение per guild или per user
- 🌐 **Мультисерверность** — поддержка серверов 1-5
- 🎨 **Красивые embed'ы** — форматированные сообщения с эмодзи и цветами
- ⚡ **Slash-команды** — современный интерфейс команд Discord
- 🎮 **Интерактивные UI** — Select Menu для удобного выбора категорий
- 🛡️ **Обработка ошибок** — полная обработка 401, 403, 429 и других ошибок
- 🔄 **Авторетрай** — автоматические повторные попытки при ошибках
- 📝 **Логирование** — подробное логирование в файл и консоль
- 🔒 **Безопасность** — ключ показывается только один раз при установке

## 🚀 Установка

### Предварительные требования

- Python 3.9 или выше
- Discord Bot Token
- API ключ Majestic RP

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/BLACKSPANIEL/Market-stats.git
cd Market-stats
```

### Шаг 2: Установка зависимостей

```bash
pip install -r requirements.txt
```

### Шаг 3: Настройка переменных окружения

Скопируйте файл `.env.example` в `.env`:

```bash
cp .env.example .env
```

Отредактируйте `.env` и добавьте ваш Discord токен:

```env
DISCORD_TOKEN=your_discord_bot_token_here
```

### Шаг 4: Создание Discord бота

1. Перейдите на [Discord Developer Portal](https://discord.com/developers/applications)
2. Создайте новое приложение
3. Перейдите во вкладку **"Bot"** и создайте бота
4. Скопируйте токен бота в `.env` файл
5. Включите **Message Content Intent** (если нужно) и **Server Members Intent**
6. Перейдите во вкладку **"OAuth2"** → **"URL Generator"**
7. Выберите scope `bot` и `applications.commands`
8. Выберите permissions:
   - `Send Messages`
   - `Embed Links`
   - `Use Slash Commands`
9. Скопируйте сгенерированную ссылку и добавьте бота на сервер

### Шаг 5: Запуск бота

```bash
python main.py
```

## 📖 Команды

### 🔑 Управление API ключом

#### `/setkey` — Установить API ключ
Установите API ключ от Majestic RP для доступа к статистике.

**Параметры:**
- `api_key` (обязательный) — ваш API ключ
- `personal` (опциональный) — установить ключ лично (только для вас) или для всего сервера

**Примеры:**
```
/setkey api_key=your_api_key_here
/setkey api_key=your_api_key_here personal=True
```

**Примечание:** 
- Для установки серверного ключа нужны права администратора
- ⚠️ Ключ показывается только один раз при установке! Сохраните его.

#### `/mykey` — Проверить API ключ
Проверяет, установлен ли у вас API ключ (без показа самого ключа).

**Пример:**
```
/mykey
```

#### `/removekey` — Удалить API ключ
Удаляет сохраненный API ключ.

**Параметры:**
- `personal` (опциональный) — удалить личный ключ или серверный

**Примеры:**
```
/removekey
/removekey personal=True
```

### 📊 Статистика маркетплейса

#### `/market` — Статистика маркетплейса (основная команда)
Показывает статистику маркетплейса с интерактивным Select Menu для выбора категории.

**Параметры:**
- `category` (опциональный) — категория (vehicles, items, houses, apartments, warehouses, offices, clothes)
- `server` (опциональный) — номер сервера (1-5)

**Примеры:**
```
/market
/market category:vehicles
/market server:2
/market category:houses server:3
```

**Особенности:**
- Если категория не указана — показывается Select Menu для выбора
- Если категория указана — сразу показывается статистика
- Интерактивный Select Menu с эмодзи

#### `/stats` — Статистика с выбором категории
Показывает статистику с интерактивным Select Menu для выбора категории.

**Параметры:**
- `server` (опциональный) — номер сервера (1-5)

**Пример:**
```
/stats
/stats server:2
```

#### `/listings` — Показать объявления
Показывает актуальные объявления категории с пагинацией.

**Параметры:**
- `category` (обязательный) — категория
- `server` (обязательный) — номер сервера
- `page` (опциональный) — номер страницы (по умолчанию 1)

**Пример:**
```
/listings category:houses server:2 page:1
```

## 🔧 Конфигурация

### config.py
- `DISCORD_TOKEN` — токен Discord бота
- `MAJESTIC_API_BASE_URL` — базовый URL API Majestic RP
- `CATEGORIES` — словарь категорий маркетплейса (7 категорий)
- `SUPPORTED_SERVERS` — список поддерживаемых серверов (1-5)
- `DEFAULT_SERVER` — сервер по умолчанию (1)
- `COLORS` — цвета для embed'ов (по категориям)
- `API_TIMEOUT` — таймаут запросов (30 сек)
- `API_RETRY_ATTEMPTS` — количество повторных попыток (3)
- `LISTINGS_PER_PAGE` — объявлений на странице (10)

### Хранилище API ключей
API ключи хранятся в `data/keys.json` в формате:
```json
{
  "guild_123456789": "server_api_key",
  "user_987654321": "personal_api_key"
}
```

**Приоритет ключей:** personal key > guild key

**Безопасность:**
- Ключ показывается только один раз при установке (в ephemeral сообщении)
- Файл `keys.json` содержит чувствительные данные
- Личные ключи видны только пользователю, который их установил
- Атомарная запись файла (через .tmp файл)

## 🛡️ Безопасность

- API ключи хранятся локально в JSON файле
- Серверные ключи требуют прав администратора
- Личные ключи видны только пользователю, который их установил
- Ключ показывается только один раз при установке через `/setkey`
- Файл `.env` не должен коммититься в репозиторий
- Файл `data/keys.json` содержит чувствительные данные
- Валидация формата ключа перед сохранением

## 📝 Логирование

Бот ведет подробное логирование:
- **Файл:** `bot.log` (в кодировке UTF-8)
- **Консоль:** вывод в stdout

Логи содержат:
- Время события
- Уровень логирования
- Имя модуля
- Сообщение
- Stack trace для ошибок

## ⚠️ Обработка ошибок

Бот обрабатывает следующие ошибки:
- ❌ Неверный API ключ (401 Unauthorized)
- 🚫 Доступ запрещен (403 Forbidden)
- ⚠️ Превышен лимит запросов (429 Rate Limit)
- 🔍 Ресурс не найден (404 Not Found)
- 🔧 Ошибка сервера (5xx Server Error)
- 🌐 Ошибки сети
- 💥 Неожиданные ошибки

Все ошибки логируются и показываются пользователю в виде понятных embed'ов.

## 🔍 Troubleshooting

### Бот не запускается
- Проверьте, что `DISCORD_TOKEN` указан в `.env`
- Убедитесь, что токен действителен
- Проверьте логи в `bot.log`

### Команды не появляются
- Убедитесь, что бот имеет права `applications.commands`
- Перезапустите бота для синхронизации команд
- Проверьте, что бот добавлен на сервер

### Ошибка "API ключ не найден"
- Используйте `/setkey` для установки ключа
- Убедитесь, что ключ действителен
- Используйте `/mykey` для проверки

### Ошибка "Неверный API ключ"
- Проверьте правильность ключа
- Используйте `/removekey` и `/setkey` для обновления
- Убедитесь, что ключ не истек
